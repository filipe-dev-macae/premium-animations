# Scroll Design

Scroll motion should support story, comparison, depth, or progressive disclosure. It should never make users fight the page.

## Use Scroll Motion For

- Product walkthroughs.
- Case-study reveals.
- Timelines.
- Feature comparisons.
- Sticky visual explanations.
- Editorial image reveals.

## Avoid Scroll Motion For

- Checkout.
- Complex forms.
- Dense dashboards.
- Admin tables.
- Critical pricing comparison.
- Documentation reading flows unless subtle.

## Rules

- Keep text stable.
- Avoid scroll fatigue.
- Avoid pinning ordinary content.
- Keep horizontal scroll rare and justified.
- Provide mobile fallbacks.
- Respect keyboard navigation.
- Respect reduced motion.
